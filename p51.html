  • table要素のborderや背景色、文字色の装飾

h2要素にidを設定し、この色を変化させる。table要素をborder属性を消して罫線・セルを工夫する。
CSSでどんな指定をしたか説明する。ルールセットの記述内容、その他具体的効果など。

スタイルシート

cssの指定

link要素のrel要素にstylesheetを設定し、 href属性として適用する外部スタイルシートのURLを示すことで Webページにスタイルシートを関連づけて表示できる。
<link rel="stylesheet" herf="<指定するファイルのURLまたはパス名>">

cssの書式

<セレクタ> {<プロパティ>: <値>;
<プロパティ>: <値> ...}
のような書式をとる。
セレクタはスタイルを適用する対象となるHTML構文木の部分木を指す（例えば、pならすべてのp要素）。 プロパティと値の組を宣言と呼び、セレクタ指定部分に対して実際に適用するスタイルを指定する。 セミコロンで区切ることで複数の宣言を同時に指定することができる。
例
* {
color: #5555ff;
border: thin solid #ffffff;
background-color: #dddddd;
}
body {
background-color: #90b090;
}
td.absentee {
font-size: x-small;
background-color: #f0a0a0;
text-align: center;
}
このようにtd.absenteeとすればtd要素のうちclass="absentee"の要素にのみスタイルを指定する。

クラスセレクタは先頭のピリオドに続けてclass属性名を記述する。
IDセレクタは先頭がハッシュ記号#である。
複数のセレクタが同一のHTML部分木を指し示すとき、どの宣言を優先するかをきめる要因の一つにセレクタの詳細度パラメタがある。 詳細度はそのセレクタを構成する各セレクタの詳細度の総和で、高い点数のものほど優先される。

セレクタの詳細度

要素型、疑似要素1td, ::first-letter
クラス、属性など10.className, [href="..."]
ID100#idName

border-style : 境界線の形状。ブラウザによって対応が異なる。none,hidden,dotted,solid,double,groove,ridge,inset,outset など
border-width : 境界線の太さ。thin,medium,thick,長さ など
border-color : 境界線の色。色で指示する。

JavaScript

処理モデル

実行環境がWebブラウザ上にある場合、WindowやHTMLdocumentなどを組込みオブジェクトとして持つ。 HTMLドキュメントは木構造で表現され、DOM（Document Object Model）に基づいた表現として取り扱われる。 また、CSS要素にもアクセス可能である。 document treeやCSStreeにアクセスし、要素の追加、削除、更新や属性の追加、属性値の変更などを行うことで、 Webコンテンツに対して非常に広範囲の変更を加えることができる。利用者とのインタラクション目的に利用されることを企図して設計されているため、 プログラムの起動トリガーとしてのイベント駆動モデルで実行される。
インターネット環境での利用を前提としているため、プログラムを取得したドメインのサーバーとの通信しか許可されず、 実行中のローカルマシンに対するファイルアクセスも許されない。

HTMLとの関連付け

webコンテンツではHTML内に関連付けるJavascriptプログラムファイルへのURLを記述することで、ブラウザにそのプログラムファイルの読み込みを指示する。
JavascriptプログラムをHTMLファイルと関連付けるためにはHTMLのscript要素をhead要素化に配置し、外部ファイルを指定することが一般的である。
例<script src="指定したいURL"></script>
script要素は要素内容としてスクリプトプログラムを記述することも可能である。
src属性には実際に読み込むべきJavascriptファイルへのフルURL、絶対パス、相対パスのいずれかを記述する。 通常は同一サイト内のファイルを読み込むため、絶対パスか相対パスで指定する。 script要素は読み込みたいリソースの数に応じて任意の個数を記述できる。Javascriptでは読み込まれた順に実行されていく。 ここで、指定したJavascriptプログラムでは特定のHTML要素を捜査対象とするため、操作したい当該HTML要素には、class,id属性により識別子を割り当てておくと便利である。

windowオブジェクト

windowはWebブラウザのウインドウ自体を示すオブジェクトで、Web ブラウザ上で実行される際には、実行環境が事前に準備して利用可能な状態として提供される。 オブジェクトは、状態を表す属性情報を保持する容れ物としてプロパティと呼ばれるものを任意の個数持てる。 プロパティには、数値や文字列、他のオブジェクト、メソッドなどを保存できる。 各プロパティは識別のため名付けることができて、これをプロパティ名などと呼び、通常はこの名前で保存容器としてのプロパティを指定する。
たとえば、abc と名付けられたオブジェクトに def と ghi という 2 つのプロパティがあったとすると、 それぞれ、abc.def、abc.ghi のようにオブジェクト名とプロパティ名をドット（.; ピリオド）でつないで記述する。 プロパティは空の場合もあり、数値や文字列などの単純なデータらしいものが入っていることもある。 また、他のオブジェクトやメソッドも入れられる。このような容れ物（箱）として見た場合、プロパティは変数と呼ばれる。 中身が変わることのあるデータ（数値だけに限らない）、という意味合いである。 気をつけなければならないのは、他のオブジェクトやメソッドは、大きすぎて直接プロパティには入らない点である。
たとえばメソッドは、手順書のようなものであり、長い紙に多数の手順を連ねたようなものであると言える。 これは、小さな容器であるプロパティ（変数）にはそのまま入らない。 そこで、このようなものは、本体（ここではメソッド定義；手順書）は棚にしまっておき、 変数には、「棚のどこどこの位置に置いてあるもの」というような小さな紙片だけを入れておくような処理が行われる。 この格納位置を指し示していることを「参照」と呼ぶ。 オブジェクトのプロパティにメソッド定義を保存（代入）してやると、そのプロパティ名を使って、 このメソッドを呼び出す（実行）することができる。JavaScript の処理系は、プロパティに保存された参照を使って、 メソッド定義の本体を見つけ、それをプログラムとして実行するのである。

DOMモデル

HTMLコンテンツはJavaScriptではDOM（Document Object Model）と呼ばれるデータ構造でモデル化され操作できる。 DOMはHTMLの木構造を表現したデータモデルである。HTMLの要素をJavaScriptのオブジェクトとして表現したものである。 DOMはJavaScript言語自体の機能ではないが、W3Cにおいて標準化され、Webブラウザ上で機能するJavaScript処理系では共通して利用できるようになっている。 DOMで表現されたHTML木は木構造をJavaScriptのプログラムから扱える入れ子構造のオブジェクトになっている。 実際には入れ子構造のオブジェクトに直接アクセスするというより、木構造を考慮したAPIによって操作することになる。 このDOMを操作する一連のAPIはDOMインタフェースと呼ばれる。

タイマ

Javascriptのタイマには2種類ある。

  1. 設定時間経過後に処理させる
  2. 設定時間毎に繰返し処理させる

この2種類である。処理内容を登録しておき、設定時間経過後にその処理が呼び出されることになる。 待機中のタイマ処理を取り消すにはあらかじめ設定methodを呼び出した際の戻り値を適当な変数に割り当てておく必要がある。
